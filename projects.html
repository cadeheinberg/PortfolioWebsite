Java Plugin

  • Resources Used: Minecraft, Java, Spigot API, MySQL, Ubuntu
  • Source Code: github
Minecraft is a popular Java based video game. Since 2020 I have been making Java plugins for a Minecraft server that I host...
Most Minecraft Plugins including mine are built upon the Spigot API/Library. I also use a MySQL server that stores the player stats, leaderboards, achievements, etc. I have been hosting the server from an ubuntu machine at my house, but I have also periodically hosted it on Google Cloud. Going from a fresh Ubuntu install to a fully functioning Minecraft + MySQL server is quite the endeavor. But in the end I Port Forwarded the server and have been able to join SSH to the console remotely.

IMDb Interactive

  • Resources Used: Tableau, IMDb Free Datasets, SQLite, Python
  • Live Site (desktop only): IMDb Interactive
An interactive web based visualizer for IMDbs free dataset on Movies, TV shows, actors, producers, writers, etc...
Step 1: I downloaded 7 tab separated files from the IMDb dataset containing various information on movies and tv shows
Step 2: IMDB uses “\N” for NULL values in their .tsv files, so I wrote a python script to change “\N” characters to be “null”.
Step 3: I used GitBash to execute various SQLite commands to convert the tsv files into tables inside of a movies.db file allowed me to set “null” strings to NULL inside .db file. See the source code above for iist of these commands
Step 4: I made a Jupter Notebook envirnoment and loaded the .db file into SQLite. I performed an extensive amount of queries to produce new tables from the original ones that held data in an optimized form. This would make graphing and interactions faster.
Step 5: I used DB Browser to convert the new tables to .csv files to load into Tableau.
Step 6: I used Tableau to create several sheets visualizing the data in the .csv files. I created a dashboard to display all of these sheets in one place and created calculated fields and custom filters to allow interactions between each visualization based on user inputs.
